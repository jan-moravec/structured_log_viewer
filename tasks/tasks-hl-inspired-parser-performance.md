On miss the failure path emits a single `qCritical()` line listing every searched candidate and the supported invocation patterns (mirroring the `FAIL(...)` message in `common.cpp` 196–203), then `QApplication::exit(1)` — no modal dialog, no hang, regardless of platform plugin.

### Notes

- **Build & test commands.** `cmake --preset local && cmake --build --preset local --target tests benchmarks`. Run unit tests with the produced `tests.exe` (Catch2; CTest also wraps it via `catch_discover_tests`). Run benchmarks with the **separate** `benchmarks.exe` binary, e.g. `benchmarks.exe "[.][benchmark][json_parser][large]"` (and `[wide]` / `[stream_to_table]` / `[allocations]` / `[no_thread_local_cache]` / `[get_value_micro]` / `[cancellation]` per the per-task brief). The split was introduced during the task 1.0 capture pass because, when `benchmark_json.cpp` lived inside `tests.exe`, every Catch2 random-order test pass dragged the long-running parse fixtures into the unit-test runtime budget. Task 1.0 also fixed a latent infinite loop in `InitializeTimezoneData()` (on Windows, `std::filesystem::path("C:\\").parent_path()` returns `C:\\` itself, so a missing `tzdata` ancestor would loop forever): the walk-up loop now terminates at the filesystem root and, on miss, emits a Catch2 `FAIL` naming the searched CWD and the supported invocation patterns. Callers are expected to run the binary either via `ctest --preset local` or from a directory whose ancestor chain contains the staged `tzdata/`. Both issues are addressed and unit tests now finish in ~3 s under `ctest --preset local`.
- **Per the PRD §7.4, every commit in this PRD must include a before/after `MB/s` line in its commit message** for `[large]`, `[wide]`, and `[stream_to_table]`, plus the per-stage breakdown emitted by parent PRD §4.7.2 (Stage B utilisation is the headline). Acceptance gate G4 is ±3 % per change or a documented architectural justification in the commit message.
- **Each parent task is its own commit** per the PRD's §4 contract ("Each requirement is a separate PR"). The parent PRD's task list ran on the same branch with one commit per parent task; this PRD continues that convention.

- **`tsl::robin_map`, `oneTBB`, `simdjson`, `Catch2` are already linked**; tasks 1.0, 2.0, 3.0, 5.0 require **no** CMake dependency additions. Only task 4.0 (conditional) potentially adds `xxhash-cpp` / `xxHash`.
