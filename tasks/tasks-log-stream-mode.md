### Notes

- Library tests live in `test/lib/` (Catch2). App smoke tests live in `test/app/` (offscreen Qt). Benchmarks share the Catch2 setup and are tagged (e.g. `[large]`, `[wide]`, `[allocations]`, `[cancellation]`); the new latency benchmark is tagged `[stream_latency]`.
- Run library tests via the project's existing CTest preset (`ctest --preset release` or the equivalent in `CMakePresets.json`); benchmarks are filtered by tag, e.g. `ctest -L benchmark` (see `test/lib/CMakeLists.txt`).
- The static-open path goes through an mmap'd `LogFile` parsed by `JsonParser::ParseStreaming` over the TBB pipeline in `library/src/parser_pipeline.cpp`. Stream Mode reuses everything **above** the source layer (`KeyIndex`, `LogConfiguration` / `LogConfigurationManager`, `LogTable` `BeginStreaming` / `AppendBatch` / `EndStreaming`, `StreamingLogSink`, `WorkerScratchBase` / `BuildTimeColumnSpecs`) and replaces only the byte producer below it.

  - [ ] 3.7 Implement rotation detection in branch order (PRD 4.8.6 i / ii / iii); the first match wins:
    - **(i) File-identity changed** — `FromPath(path)` ≠ the originally-opened identity → close the old handle, open the new path, seek to offset 0, resume.
    - **(ii) Path missing** — `stat`/`GetFileAttributes` says `ENOENT` / `ERROR_FILE_NOT_FOUND` → enter the "waiting" state, retry every 250 ms; on first successful re-open install the new handle, seek to offset 0, resume; if the path reappears with non-zero size between two polls, branch (i) wins on the next poll.
    - **(iii) Size shrunk on the open handle** — `fstat`/`GetFileSizeEx` < the last-known read offset, *and* (i) did not match → `seek(0)` on the existing handle and resume; **do not** re-open (the producer may rely on the share mode).
