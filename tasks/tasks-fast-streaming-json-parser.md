# Tasks: High-Performance Streaming JSON Log Parser

Source PRD: `tasks/prd-fast-streaming-json-parser.md`

## Relevant Files

### New library files

- `library/include/loglib/key_index.hpp` — public `KeyIndex`/`KeyId`/`kInvalidKeyId` types (req. 4.1.1, 4.1.2/2a).
- `library/src/key_index.cpp` — `KeyIndex` implementation (TBB `concurrent_hash_map` + `std::deque<std::string>` reverse storage; transparent `string_view` hashing).
- `library/include/loglib/streaming_log_sink.hpp` — `StreamingLogSink` and `StreamedBatch` (req. 4.3.25).
- `library/src/buffering_sink.hpp` / `library/src/buffering_sink.cpp` — internal `BufferingSink` used by the legacy `Parse(path)` API and by tests (req. 4.3.25).
- `library/src/line_source.hpp` / `library/src/mmap_file_line_source.hpp` — internal `ILineSource` interface and the only shipped impl (req. 4.4.31–33).
- `test/lib/src/test_key_index.cpp` — unit tests (sequential and concurrent) for `KeyIndex` (req. 4.1.16).

### Existing library files modified

- `library/include/loglib/log_line.hpp`, `library/src/log_line.cpp` — indexed storage, fast/slow accessors, `LogValue` extension, `AsStringView`/`HoldsString`/`ToOwnedLogValue`/`LogValueEquivalent`/`LogValue::TrustView`, drop `LogMap` (req. 4.1.5–4.1.10, 4.1.16).
- `library/include/loglib/log_file.hpp`, `library/src/log_file.cpp` — own `mio::mmap_source`, drop `std::ifstream`, switch to `std::vector<uint64_t>` line offsets, add `Data()`, `ReserveLineOffsets()`, `HintSequential` helper, `LogFileReference::SetLineNumber` (req. 4.1.6a/b, 4.1.9a, 4.2.18).
- `library/include/loglib/log_data.hpp`, `library/src/log_data.cpp` — own canonical `KeyIndex`, `AppendBatch`, `TimestampsAlreadyParsed` flag; `Merge` keeps remap path (req. 4.1.3, 4.1.4, 4.2.21).
- `library/include/loglib/log_table.hpp`, `library/src/log_table.cpp` — internal `column → KeyId` cache, append-only column contract, `AppendBatch(StreamedBatch)`, `LastBackfillRange`, mid-stream timestamp back-fill (req. 4.1.11–13a/b).
- `library/include/loglib/log_parser.hpp` — `ToString(const LogLine&)` signature (req. 4.1.14).
- `library/include/loglib/json_parser.hpp`, `library/src/json_parser.cpp` — `JsonParserOptions`, `ParseStreaming`, three-stage `tbb::parallel_pipeline`, opt-in per-worker caches, padding contract, `field.key()` fast path, `string_view`-preferring value emission, `BufferingSink`-backed legacy `Parse` (req. 4.1.15/15a/15b, 4.2.17–22c, 4.3.25–26a).
- `library/src/log_processing.cpp` — replace direct `std::holds_alternative<std::string>`/`std::get<std::string>` with `loglib::AsStringView` (req. 4.1.6, 4.2.21 back-fill helper reuse).

### Existing app files modified

- `app/include/log_model.hpp`, `app/src/log_model.cpp` — `BeginStreaming`/`AppendBatch`/`EndStreaming`, owned `QtStreamingLogSink`, `lineCountChanged`/`errorCountChanged`/`streamingFinished` signals, `Clear()` cancellation (req. 4.3.27, 4.3.30).
- `app/include/qt_streaming_log_sink.hpp` (new), `app/src/qt_streaming_log_sink.cpp` (new) — `QObject` bridging adapter with generation id and owned `std::stop_source` (req. 4.3.28).
- `app/include/main_window.hpp`, `app/src/main_window.cpp` — switch `OpenFilesWithParser`/`OpenFileInternal` to streaming; status-bar progress via signals; gate config UI between begin/end stream (req. 4.3.29).

- Run unit tests with `ctest --preset <preset>` (or directly via the Catch2 binary). Benchmarks are gated behind `[benchmark]` tags and are opt-in.
- Update this task list as work progresses (`- [ ]` → `- [x]`) after **each** sub-task, not just after each parent task.
- Keep commits small and self-contained per sub-task to make review tractable; the parent-task boundaries are also good PR-split candidates if a single PR becomes unwieldy.
- For PRD requirement IDs referenced below (e.g. "req. 4.1.6a"), see `tasks/prd-fast-streaming-json-parser.md`.
