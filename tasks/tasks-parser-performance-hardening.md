### Notes

- **Build & test commands.** `cmake --preset local && cmake --build --preset local --target tests`. Run unit tests with the produced `tests.exe` (Catch2). Benchmarks: `tests.exe "[.][benchmark][json_parser][large]"` (and `[wide]` / `[stream_to_table]` once §4.7 / §4.2a land).
- **Per the PRD §7.4, every commit in this PRD must include a before/after `MB/s` line in its commit message.** Methodology: pre-warm once, then take the warm-up run + Catch2 mean of 100 on the `[large]` fixture (and `[wide]` / `[stream_to_table]` once those exist). Acceptance gate G4 is ±3 % per change or a documented architectural justification in the commit message.
- **Each parent task is its own commit** per the PRD's §4 contract ("Each requirement is a separate PR"). Order within a priority tier may shift if dependencies make it natural, but P1 work (1.0–3.0) lands before P2 (4.0–5.0), which lands before P3 (6.0–9.0).
- **Task 10.0 is conditional** — only execute its sub-tasks if the post-P3 numbers leave Stage B utilisation < 70 % or wall-time clearly bottlenecked outside the addressed hot paths. Otherwise mark its sub-tasks as `cancelled` with a one-line justification.
- **All new options are additive** (`JsonParserOptions` only). No public API signatures change (cf. PRD N1).
- **simdjson is pinned at v4.6.3** in `cmake/FetchDependencies.cmake`, well above the 3.6 cutoff in §4.4.1 — so the SWAR fallback is dead code and task 5.0 goes straight to `field.escaped_key()`.
- **`tsl::robin_map` is already linked** as a dependency (see `library/CMakeLists.txt`), so no CMake changes are required for tasks 1.0 or 2.0 (Route C).
